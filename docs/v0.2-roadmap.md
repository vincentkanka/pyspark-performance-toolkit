# SparkScope v0.2 Roadmap: Join and Partition Analyzer

## Objective

SparkScope v0.2 expands beyond static Spark configuration review by introducing workload metadata analysis for joins and partitioning.

The goal is to help data engineers identify common Spark performance risks before production execution, including unnecessary shuffle joins, missed broadcast opportunities, large unpartitioned datasets, and partitioning mismatches.

## Planned Capabilities

### 1. Table Metadata Analysis

SparkScope will evaluate table-level metadata such as:

- Table name
- Approximate size in MB
- Approximate row count
- Partition columns

Potential findings:

- Large table has no partition columns
- Very small table may be suitable for broadcast joins
- Extremely large table may require partitioning review

### 2. Join Metadata Analysis

SparkScope will evaluate join-level metadata such as:

- Left table
- Right table
- Join type
- Join keys
- Table size comparison

Potential findings:

- Broadcast join opportunity detected
- Large-to-large join may cause expensive shuffle
- Join key may not align with partition columns
- Missing join key metadata

### 3. Severity Model

Findings will continue to use the existing severity levels:

- HIGH
- MEDIUM
- LOW

Severity will be based on estimated performance risk, metadata completeness, and likely shuffle impact.

## Example Input

```json
{
  "tables": [
    {
      "name": "transactions",
      "size_mb": 50000,
      "row_count": 200000000,
      "partition_columns": ["transaction_date"]
    },
    {
      "name": "customers",
      "size_mb": 50,
      "row_count": 100000,
      "partition_columns": []
    }
  ],
  "joins": [
    {
      "left_table": "transactions",
      "right_table": "customers",
      "join_type": "inner",
      "join_keys": ["customer_id"]
    }
  ]
}

## Example Findings
MEDIUM: Broadcast join opportunity detected
The table customers is small relative to transactions and may be suitable for broadcast join evaluation.

MEDIUM: Join key does not align with partition columns
The join key customer_id is not present in the partition columns for transactions.
